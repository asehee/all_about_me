# Working Rules

This repository follows a frontend-first workflow for UI tasks.

## Frontend Flow (Required)

For feature work that includes UI:

1. Publishing/layout first
2. Then state/event wiring
3. Then API integration
4. Then polish and QA

Do not start API wiring before the publishing baseline is approved.

## Publishing Baseline Checklist

A UI task is considered "publishing complete" only when all items below are done:

- Visual hierarchy and spacing are defined.
- Desktop and mobile layouts are both implemented.
- Alignment/line-height/baseline issues are fixed.
- Interactive states exist (hover, focus, active, disabled as needed).
- Color/typography are consistent with current product mood.

## Collaboration Preference

When the user asks for UI changes, propose or implement publishing-first changes before data/model logic unless explicitly requested otherwise.
